# Sqlet Cli

Run Refinery migrations via cli.

## Installation

The binary name for `sqlet_cli` is `sqlet`.

**[Archives of precompiled binaries for sqlet are available for Windows,
macOS and Linux.](https://github.com/rust-db/sqlet/releases/)** Linux and
Windows binaries are static executables. Users of platforms not explicitly
mentioned below are advised to download one of these archives.

### Debian/Ubuntu

If you're a **Debian** user (or a user of a Debian derivative like **Ubuntu**),
then sqlet can be installed using a binary `.deb` file provided in each
[sqlet_cli release](https://github.com/rust-db/sqlet/releases/).

```sh
$ curl -LO https://github.com/rust-db/sqlet/releases/download/0.8.4/sqlet_0.8.4_amd64.deb
$ sudo dpkg -i sqlet_0.8.4_amd64.deb
```

### Arch Linux

If you're an **Arch Linux** user, then you can install
the [`sqlet_cli`](https://aur.archlinux.org/packages/sqlet_cli) package from AUR:

```sh
$ yay sqlet_cli
```

### NixOs

If you're a **Nix** user, you can install sqlet via
the [`sqlet-cli`](https://github.com/NixOS/nixpkgs/blob/master/pkgs/development/tools/sqlet-cli/default.nix) package:

```sh
$ nix-env -iA sqlet-cli
```

### Cargo.

If you're a **Rust programmer**, `sqlet_cli` can be installed with `cargo`.


```sh
$ cargo install sqlet_cli
```

## Usage

Setup your database type and access credentials with `setup`.

```sh
$ sqlet setup
```

After that, just run your migrations giving your config file with `-c` flag (by defaults it is the `sqlet.toml` generated by the setup) and migrations dir with `files -p $dir`.

```sh
$ sqlet migrate -c sqlite_sqlet.toml -p ./sql_migrations
```

### Running via database uri

To run migrations from a database [uri](http://www.postgresql.org/docs/current/static/libpq-connect.html#LIBPQ-CONNSTRING) (like: postgres://user_name:passwd@hostname:5432/myDB ) stored in an environment variable DB_URI.

```sh
$ sqlet migrate -e DB_URI -p ./sql_migrations
```

This option is also useful when running sqlet inside a docker container, where you usually have the db connection info stored as an environment variable.

For more info and migration options run.

```sh
$ sqlet migrate --help
```

### Generating migrations

To generate a new migration, use the `generate` command along with a name for your migration. This will create a new migration directory with up.sql and down.sql files.

```sh
$ sqlet generate create_users_table
```

By default, migrations are created in the ./migrations directory. To specify a different directory, use the -p flag:

```sh
$ sqlet generate -p ./my_migrations create_users_table
```

If the specified migrations directory doesn't exist, it will be created automatically.

This will create a timestamped migration directory using UTC time (e.g., 20250524_222017_create_users_table) containing up.sql and down.sql files ready for you to add your SQL migration code.
